TL;DR:

* [Much bloat](http://spinroot.com/gerard/pdf/Code_Inflation.pdf) - Note author and also source.
* Linux API is organic and sometimes not designed very well.
  * namespaces - provide no security.
  * Syscalls to get random numbers: [Linux](http://man7.org/linux/man-pages/man2/getrandom.2.html) vs [OpenBSD](http://man.openbsd.org/arc4random)
  * kqueues not present in Linux.
* "Not invented here" syndrome
* There are alternatives where people try to learn from history.
* Linux has an advantage though: portability: OS boundary ends at Kernel level. This leads to alternative libcs (musl, ulibc, etc). Easy virtualize.
